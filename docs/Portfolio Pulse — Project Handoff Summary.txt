Portfolio Pulse — Project Handoff Summary
Original Brief
Build a production-quality, AI-powered portfolio intelligence platform called "Portfolio Pulse" for personal use and portfolio showcasing. Premium dark-themed finance UI (Bloomberg-modernized, Linear/Vercel-inspired), built primarily through AI-assisted development by a non-experienced developer.
Core product: Monitors stocks (NSE/BSE), crypto, mutual funds, market/geopolitical events, and news. Uses AI to determine relevance to the user's specific holdings and generates impact alerts with explanations. Explicitly NOT a trading platform, broker integration, prediction system, or financial advice tool — monitoring, filtering, relevance analysis, and summaries only.
Tech Stack (locked in)
* Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS, Framer Motion, Recharts, shadcn/ui
* Backend: Firebase Auth, Firestore (region: asia-south2 Delhi), Firebase Admin SDK
* AI: Gemini API — currently gemini-2.5-flash via direct REST /v1/ endpoint (not SDK, not v1beta)
* OCR: Gemini Vision (base64 inline images, never persisted)
* Hosting: Vercel (not yet deployed)
* Data sources: Yahoo Finance (NSE/BSE prices), CoinGecko (crypto prices), RSS feeds + NewsAPI (news)
Critical Architecture Decisions
1. No Firebase Storage — Delhi Firestore region isn't eligible for no-cost Storage tier. Screenshots go to Gemini Vision as base64, never persisted.
2. Two-stage news filter — Stage 1: cheap ticker-dictionary string matching (src/lib/ticker-dictionary.ts). Stage 2: Gemini AI analysis only on survivors. Keeps AI costs near zero.
3. OCR review required — extracted holdings always shown in editable table before Firestore commit. Manual "add holding" fallback always available.
4. Soft deletes — deletedAt: Timestamp | null on holdings/watchlist, never hard delete.
5. Server/Admin SDK split — src/lib/firebase/admin.ts (server-only), src/lib/firebase/client.ts (browser). Session cookie pp_session, httpOnly. Middleware does cookie-existence check only (Edge runtime can't run Admin SDK); full verifySessionCookie happens in dashboard layout.
6. Firestore Timestamps must be converted to ISO strings or null before passing from Server to Client Components — raw Timestamp objects throw "Only plain objects can be passed" errors.
7. fetchPrices() has server-only — client components must call it via /api/prices POST route.

Required Firestore Composite Indexes (all created)

* holdings (collection group): deletedAt ASC + symbol ASC
* alerts (per-user subcollection): dismissed ASC + createdAt DESC
