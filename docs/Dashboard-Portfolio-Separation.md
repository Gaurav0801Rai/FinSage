# Architectural Q&A & Design Update Plan

This document explains the mechanism behind live price fetching, how to resolve the blank pages in your local development environment, and details the plan to separate the high-level **Dashboard Summary** from a detailed **Portfolio View** grouped by asset categories.

---

## 1. How Live Prices are Fetched Without API Keys

You do not need to register or pay for a Yahoo Finance or CoinGecko API key in this project. The price fetching service in `src/lib/price-service.ts` uses public endpoints:

1. **Indian Stocks, ETFs, and Mutual Funds (Yahoo Finance)**:
   * **Endpoint**: `https://query2.finance.yahoo.com/v8/finance/chart/{symbol}?interval=1d&range=5d`
   * **How it works**: Yahoo Finance hosts a public Chart API for standard web requests. The server-only fetch mocks a browser `User-Agent` (e.g. `Mozilla/5.0...`) in the request headers to prevent Yahoo's rate limiter from blocking the bot, allowing anonymous fetching of tickers like `RELIANCE.NS` or `TCS.BO`.
2. **Cryptocurrencies (CoinGecko)**:
   * **Endpoint**: `https://api.coingecko.com/api/v3/simple/price?ids={coin_ids}&vs_currencies=usd,inr&include_24hr_change=true`
   * **How it works**: CoinGecko provides a public, unauthenticated simple price endpoint that allows up to 10-30 requests per minute on their free tier, which easily covers our needs through our 60-second in-memory server cache.

## 2. Why Alerts, News, and Watchlist Pages are Blank (and how to fix them)

During local development, these pages are empty because they depend on data generated by users or background crons:

*   **Watchlist Page**:
    *   *Why it is blank*: Watchlists are user-specific. Since your test account has not added any assets yet, the page renders an empty state.
    *   *How to fix*: Go to the `/watchlist` page in your browser, click **"Add asset"**, search for a ticker (e.g., `RELIANCE` as a stock or `BTC` as crypto), and add it.
*   **News & Alerts Pages**:
    *   *Why they are blank*: In production, Vercel triggers news ingestion and processing cron jobs automatically. Locally, these background jobs do not run. Without news documents or matched tickers, no user alerts are generated.
    *   *How to populate them locally*:
        1. Open your browser and visit `http://localhost:3000/api/cron/ingest-news`. This will fetch RSS feeds, match them against ticker dictionaries, and populate the raw `news` collection.
        2. Next, visit `http://localhost:3000/api/cron/process-news`. This sends the news to Gemini, caches the relevance analysis, and writes alert entries into your user's `users/{uid}/alerts/` subcollection.
        3. Go back to `/news` and `/alerts` to see the populated feed!

## 3. Design Separation: Dashboard vs. Portfolio

Currently, the holdings table (listing amounts, quantities, and buy prices) is shown at the bottom of the Dashboard page, and clicking "Portfolio" in the sidebar takes you straight to the upload page. 

We will restructure this according to your design feedback:

```
[Sidebar Navigation]
  ├── Dashboard ──► High-level summary (P&L, return metrics, charts)
  └── Portfolio ──► Categorized holdings list (Stocks, Crypto, etc.) + Add/Upload Actions
```

### Proposed Portfolio Page (`src/app/(dashboard)/portfolio/page.tsx`)
We will create a dedicated `/portfolio` view page:
1. **Category Groups**: Group all user holdings by their asset type:
   * **Equities (NSE/BSE)**
   * **Cryptocurrencies**
   * **Mutual Funds & ETFs**
2. **Detailed Metrics**: Each group will render a holdings table with detailed values:
   * Symbol & Name
   * Quantity held
   * Average Buy Price (converted to base currency if preferred)
   * Current Price & Current Market Value
   * Total P&L & Day P&L
3. **Actions**:
   * Add options in the top header: A button for **"Upload screenshot"** (routing to `/portfolio/upload`) and a button for **"Add holding manually"** (opening a form modal).
   * Actions on rows to edit or soft-delete a holding.

### Simplified Dashboard Page (`src/app/(dashboard)/dashboard/page.tsx`)
We will adjust the Dashboard view:
1. **Summary Cards**:
   * **Total Portfolio Value** & **Total Invested Capital**
   * **Daily Change** (absolute change & percentage)
   * **Annualized/All-Time Return** (StatsRow)
2. **Visualization**:
   * Donut chart showing asset allocation by type (e.g., Stock vs Crypto).
   * Area performance chart showing growth history.
3. **Removed Elements**:
   * The raw holdings list table will be removed from the bottom of the dashboard.
   * A link button saying *"View detailed holdings"* will be added to navigate directly to `/portfolio`.
